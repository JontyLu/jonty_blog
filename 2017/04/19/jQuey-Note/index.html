jQuery库延迟等待加载模式

实际应用中，一般很少使用window.onload事件来实现延迟等待加载：该事件关联的方法需要等待图片等大型元素加载完成之后才能执行：会导致：页面已经展开，图片还在缓慢加载，JavaScript交互功能假死状态（加入网速很慢），且只能执行一次；（究其原因因为jQuery的ready事件在图片加载前就完成了）

jQuery和node.js的区别

  • jQuery是运行在客户端的JavaScript库，操作DOM
  • node.js是运行在服务端的服务端程序，可以创建http连接，信息I/O等
  • 前者专注前端，后者专注于服务端，而AngularJS专注于数据绑定。
  • ASP.NET也可以支持jQuery

备注：jQuery 可以放在本地，也可以直接使用公司的CDN(内容分发网络)，因为可能用户访问的时候已经缓存在本地浏览器中了，可以加快网站的打开速度；另一个好处是节省了网站的流量带宽。

原始Ajax和jQuery中的Ajax

  1. AJAX

    AJAX是Asynchronous JavaScript and XML（异步的JavaScript和XML），不是一门新语言

    利用AJAX可以实现浏览器和服务端的数据通信，这些数据通信无需基于网页重新加载。是XMLHttpRequest、JavaScript、XML。CSS和HTML的结合。

  2. XMLHttpRequest是整个AJAX技术核心，是浏览器和服务端通信的载体

    • 负责向服务端发送请求、监听请求的状态、回调函数

  3. AJAX请求状态的变化值

    • 0：请求未初始化

    • 1：服务器连接已经建立

    • 3：请求处理中

    • 4：请求已完成，且响应已经就绪

      可以用XMLHttpRequest的status属性来获取，当大于200的时候表明服务器端成功返回正确数据
