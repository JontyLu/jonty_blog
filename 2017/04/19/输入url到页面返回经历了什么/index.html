输入url到页面返回经历了什么?

发表于 2017-04-19 | 分类于 计算机网络 | | 浏览 次

这里有一个问题，从输入url，你接受到了一个页面，但在这中间的过程，都发生了什么？

下面的内容是个人通过网上资料和自己的理解总结出的，有什么不对请指正~

废话不多说粗暴系列~

  1. DNS解析：将域名翻译成IP地址
  2. 客户端给服务器发送一个HTTP请求
  3. 服务器响应请求
  4. 浏览器解析并渲染得到的HTML

认真脸详情步骤系列

  1. DNS解析域名（域名解析协议）：将你的域名找出它的IP地址

    • 怎么找？怎么个顺序找？
      1. 先从自己的浏览器缓存找，浏览器会缓存DNS记录一段时间
      2. 再从系统缓存里找
      3. 接着把请求发给路由器，在路由器缓存找
      4. 还没有就往ISP DNS缓存找
      5. 之后ISP的DNS服务器只能开始递归搜索
        • 递归：查询交给下一个服务器，一级级传下去
        • 迭代：不知道映射的服务器把下一个服务器的IP地址发回请求查询的主机上
      6. 找到IP地址，返回给浏览器，并缓存在各个缓存里

  2. 客户端发送HTTP请求

    • 三次握手建立连接

      1. 客户端发送TCP连接请求
      2. 服务端收到请求后表示需要确认联机信息
      3. 客户端表示确认并发回（请求内容会一般会放在这里），连接建立。

    • 发送GET请求

      包含URL、编码、cookies等

  3. 服务器响应请求

    • 服务器响应请求，返回HTTP响应报文给浏览器

    • 服务器响应文件

    • 这里有个小问题：如果访问的地址不太全(少了斜杠啥的)，服务器会响应一个重定向，这样会造成一个不必要的握手了~

      Q：不重定向可以吗？–>不可以哦

      A：1. 据悉与搜索引擎排名有关，搜索引擎以为两个不同地址是两个网站(其实是一个网站)，会降低这个网站的搜索排名~。

      ​ 2. 不同的地址会造成它在地址里出现很多次

  4. 浏览器的页面渲染

    • 浏览器解析和渲染获取的的HTML(一边下载，一边加载，遇到外链就重新发送HTTP请求)
      1. 解析HTML，生成DOM树
      2. 解析CSS文件
      3. 生成渲染树
      4. 绘制渲染树

  5. 关闭连接
