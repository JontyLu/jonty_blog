散列表

也叫作哈希表

  • 哈希算法：尽可能地在数据结构中找到一个值。
  • 作用：给定一个键值，然后返回值在表中的地址
  • 优点：在散列表上插入、 删除和取用数据都非常快
  • 缺点：对于查找操作来说却效率低下， 比如查找最大最小值。
  • 冲突：由于散列函数计算出来的键值有可能相同，即有映射成同一个值的可能性

树

一个树结构包含一系列存在父子关系的节点。每个节点都有一个父节点（除了顶部的第一个节点）以及0到多个子节点。

父节点：位于树的顶部

内部节点：至少有一个子节点的节点

外部节点：没有子元素的节点

深度：节点的深度取决于它的祖先节点的数量

树的高度：取决于所有节点深度的最大值

二叉树和二叉搜索树（BST）

二叉树的节点最多只能有两个节点：左侧子节点和右侧子节点

二叉搜索树：只允许左侧节点存储比父节点小的值，右侧节点存储比父节点大或者等于的值

图

由边连接的节点，一个图G=(V,E)组成

  • V：一组顶点
  • E：一组边，连接V中的顶点

一些术语

  • 相邻顶点：由一条边连接在一起的顶点
  • 度：一个顶点的度是其相邻顶点的数量
  • 路径：顶点v1,v2…..vk的一个连续序列
  • 有向图和无向图
  • 强连通：每两个顶点之间在双向上都存在路径

  3. 关联矩阵

图的遍历

图便利算法的思想：必须跟踪每一个第一次访问的节点，并追踪哪些节点没有被完全探索

  1. 广度优先搜索

    使用队列，通过将顶点存入队列中，最先入队的顶点先被探索

    先广后深

  2. 深度优先搜索

    使用栈，通过将顶点存入栈中，顶点是沿着路径被探索的，存在新的相邻顶点就去访问

    先深后广

排序算法

冒泡排序O（n的二次方）

比较两个相邻的项，如果第一个比二个大交换他们。

运行时间最差

选择排序O（n的二次方）

找到数据结构中的最小值放在第一位，接着找到第二小的放在第二位

插入排序
归并排序O（n*log n）
快速排序O（n*log n）

二分搜索

步骤：

  1. 先排序
  2. 选择数组的中间值
  3. 如果选中值是带搜索值，则执行完毕
  4. 如果比搜索值要小，则返回1步骤并在选中值左边的子数组寻找
  5. 如果比搜索值要大，则返回1步骤并在选中值右边的子数组寻找
