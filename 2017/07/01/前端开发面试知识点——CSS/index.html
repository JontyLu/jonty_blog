BFC（Box、Formatting Context）：块级格式化上下文

概念

BFC是一个独立的渲染区域，其中

  • Box是CSS布局的基本单位，页面由很多个Box组成

  • Formatting context是页面中一块渲染区域，有自己的渲染规则，决定了子元素如何定位以及和其他元素的关系和相互作用

    BFC布局规则
    1. 内部的Box会在垂直方向，一个接一个地放置。
    2. Box垂直方向的距离由margin决定。属于同一个BFC的两个相邻Box的margin会发生重叠
    3. 每个元素的margin box的左边， 与包含块border box的左边相接触(对于从左往右的格式化，否则相反)。即使存在浮动也是如此。
    4. BFC的区域不会与float box重叠。
    5. BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。反之也如此。
    6. 计算BFC的高度时，浮动元素也参与计算

清除浮动

1、在浮动元素后使用一个空元素如\

\，并在CSS中赋予.clear{clear:both;}属性即可清理浮动。
2、给浮动元素的容器添加overflow:hidden;或overflow:auto;可以清除浮动
3、什么都不做，给浮动元素后面的元素添加clear属性。
4、利用伪类清除浮动，给浮动元素的容器添加一个clearfix的class，然 后给这个class添加一个:after伪元素实现元素末尾添加一个看不见的块元素（Block element）清理浮动。

CSS spirit图片小精灵

一种网页图片应用处理方式，把网页中一些背景图片整合到一张图片文件中，再利用CSS的

background-imge,background-repeat和background-position的组合来进行背景定位

  • 优势
    • 减少页面请求次数，提高网页加载速度
    • 减少命名干扰
  • 缺点
    • 设置背景需要得到每一个背景单元的精确位置
    • 维和图片时，最好是添加图片，而非修改已有图片
