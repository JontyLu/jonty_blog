Apply和call的用法和区别

两个方法都是在特点的作用域中调用函数

  • apply()

    • 定义：应用一个对象的方法，用另一个对象替换当前对象

    • 参数：一个是运行函数的作用域，另一个参数数组。其中第二个参数可以是Array实例，也可以是arguments对象

    • 实例

  • call()

    • 定义：调用一个 对象的方法，用另一个对象替换当前对象

    • 参数，一个还是跟apply一样——运行函数的作用域；其他的参数得逐个列举出来

    • 实例

区别

区别仅在于接受参数的方式不同，第一个参数是没有变化的，变化的是call的其余参数都需要逐个列举出来，而apply可以传入arguments或者数组。

关于this

五种使用场景

  • 作为函数调用

    this绑定全局对象（浏览器环境的全局对象为window），内部函数的内部函数的this也绑定全局对象，应该绑定到外层函数对应的对象上

  • 作为构造函数使用

    this绑定到新创建的对象

  • 作为对象方法使用

    this绑定到该对象

  • 使用apply和call调用this将会被显示设置为函数调用的第一个参数

事件冒泡机制

从目标元素开始，往顶层元素传播。途中如果有节点绑定了相应的事件处理函数，这些函数都会被一次触发。如果想阻止事件起泡，可以使用e.stopPropagation()（Firefox）或者e.cancelBubble=true（IE）来组织事件的冒泡传播。

document load和document ready两个事件的区别

页面加载完成有两种事件

  • ready

    表示文档结构已经加载完成（不包含图片等非文字媒体文件）

  • onload

    表示页面包含图片等文件在内的所有元素都加载完成了

DOM的加载步骤

  1. 解析HTML的结构
  2. 加载外部脚本和样式表文件
  3. 解析并执行脚本代码
  4. 构造HTMLDOM模型（ready事件）
  5. 加载图片等外部文件
  6. 页面加载完毕（load事件）

ready事件在DOM结构绘制完成之后就会执行，可以确保就算有大量图像文件没加载出来，JavaScript代码一样可以运行
