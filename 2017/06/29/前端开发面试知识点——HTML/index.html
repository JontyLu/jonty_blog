前端开发面试之-HTML

对语义化的理解

  • 在没有CSS时可以让页面呈现清晰的结构
  • 有利于SEO（搜索引擎优化），爬虫依赖标签
  • 方便其他设备（如盲人阅读器）的解析
  • 便于团队开发维护

Doctype作用

告知浏览器以何种模式来渲染文档

有两种模式

  • 怪异模式（浏览器自己的模式）和标准模式（W3C标准解析）
  • 如果没有声明，则不同浏览器用自己的方式来解析，会呈现不同的样式

HTML和XHMTL的区别

XHMTL：书写要求比HTML更加的严格

  • 所有的标记必须要有结束标记
  • 标签的元素和属性名字必须使用小写
  • 属性必须用引号””括起来
  • 所有属性都必须赋值
  • 图片必须有说明文字
  • 特殊符号要有编码表示

iframe

  • 优点：可以加载广告等
  • 缺点
    • iframe会阻塞页面的onload事件
    • 搜索引擎的检索程序无法解读该页面，不利于SEO
    • iframe和主页面共享连接池，而浏览器对相同域的链接有限制，会影响页面的并行加载

Label标签

该标签的for可以绑定另外一个元素的id上

src和href的区别

前者是引入，如img和script

后者是引用，如a标签和link

document.write和innerHTML

前者是重绘整个页面，后者只改变其中一个部分

HTML有哪些新特性?

  • 语义特性

    header、article、footer、nav、aside等新标签，有利于搜索引擎爬取

  • 本地存储特性

    HTML5提供了网页存储的API，离线存储包含了：

    1. 应用程序缓存
    2. 本地存储：localStorage和sessionStorage
    3. 索引数据库
    4. 文件接口

  • 设备访问特性

    1. 地理位置API：获取当前位置
    2. 媒体访问API：获取视频等
    3. 设备方向和运动API：常用于重力感应方向

  • 连接特性

    HTTP是无连接，一次请求，一次响应，长连接的效果需要用轮询的方法，但很浪费资源。

    1. Web Sockets
    2. 服务器发送事件（Server-Sent Events）

  • 网页多媒体特性

    1. Audio和Video标签
    2. Audio API：可以直接在代码中操作原始的音频流数据，对其进行任意加工再造

  • 三维、图形和特效特性

    1. SVG（可缩放矢量图形）
    2. Canvas（画布）
    3. 网页图形库（WebGL）：3D绘图标准，用于创建具有复杂3D结构的网站页面，可以设计成3D网页游戏

  • 性能与集成特性

    1. WEB Workers（网页后台任务）：解决了JS无法多线程的问题

      运行在浏览器后台的 JavaScript，独立于其他脚本，不会影响页面的性能。您可以继续做任何愿意做的事情：点击、选取内容等等，而此时 Web Workers 在后台运行。

    2. XMLHttpRequest2（新的Ajax）

请描述一下 cookies，sessionStorage 和 localStorage 的区别？

  1. 共同点

    都是保存在浏览器端，且同源

  2. 区别

    • cookie

      cookie在浏览器和服务器间来回传递，占用网络资源，而且携带数据小

    • sessionStorage

      存储量大，但因为窗口关闭前数据有效，不是永久保存的

    • localStorage

      存储量大，保存在本地，除非手动删除，否则是永久性的保存的

canvas和SVG的区别？

两者都允许在浏览器中创建图形，但在根本上是不同的

  • 前者是通过JS来绘制图形，逐像素进行渲染，依赖分辨率，不支持事件处理器，会被频繁重绘
  • 后者基于XML描述图形的语言，可以被JS操作，不依赖分辨率，支持事件处理器，适合带有大型渲染区域的应用程序如地图
