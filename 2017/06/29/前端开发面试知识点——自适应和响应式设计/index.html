前端开发面试知识点——自适应和响应式设计

发表于 2017-06-29 | 分类于 前端 | 浏览 次

自适应：Adaptive design

响应式：Responsive design

两者都是为了解决网页在不同屏幕尺寸的设备上展示的问题

概念

  • 自适应：为不同的设备监理不同的晚归，检测到设备分辨率后调用相应网页，能使网页自适应显示在不同大小终端设备上新网页设计及技术。
  • 响应式：建立一个网页，通过媒体查询，流式布局，基于内容的断点等技术来改变网页的大小以适应不同分辨率的屏幕

设计

响应式设计

  1. 三个要素

    • 流式布局或者弹性布局

      点击此处跳转详情

    • 流式图像：创建可伸缩图像

      • 可伸缩的图像要省略width和height

      • background-size可以实现背景图像的缩放

      • video、embed、object{max-width:100%}可以使视频和其他媒体变成可缩放的。

    • 媒体查询

      • 一般语法

        @media logic type and(feature:value) and(feature:value)…{…}

        • type取screen、print等；
        • logic取only或者not；
        • feature的表示如下：min-width、max-widthwidth、heightdevice-width、device-heightorientation（方向）、aspect-ratio（高宽比）、color（彩色）、monochrome（单色）、color-index（颜色数）、resolution（分辨率）、scan（扫描）、grid（栅格）、-webkit-device-pixel-ratio-moz-device-pixel-ratio

        ​

      根据不同设备的特点分别设计样式

      如果设置max-width，从大往下写，min-width反之

      ​

  2. 定义视觉区

    视觉区（viewport）指的是浏览器的客户区，即显示页面的区域，媒体查询的width就是指的视觉区的宽度。device-width指的是屏幕的宽度。

    桌面电脑的宽度（width）就是屏幕宽度（device-width），而平板和手机会把width扩大几倍，比如说，iPhone的浏览器的视觉区width默认是980px，而皮屏幕宽度实际为320px。

    如果页面设计通过媒体查询考虑了智能手机的情况，则直接使用屏幕宽度作为width，即

    width = device-width表示屏幕宽度采用设备宽度，即采用真实的屏幕宽度，initial-scale表示初始不缩放（即缩放比例为100%）

  3. 关于meta标签的viewpoint属性

    关于viewpoint属性

    width 控制viewpoint的宽度，可以是固定值，也可以是device-width，即设备的宽度
    height 高度
    initial-scale 控制初始化缩放比例，1.0表示不可以缩放
    maximum-scale 最大缩放比例
    minimum-scale 最小缩放比例

    user-scalable=0 是否允许用户调整缩放，如果设置为yes则允许用户对其改变，反之则为no，如果设置为no，那么minimum-scale和maximum-scale都将被忽略，用为根本不可能缩放。所有缩放的值都必须在0.01-10的范围之内。

    可见如果不定义viewpoint的话，页面宽度以屏幕分辨率为基准，而设置以后可以根据设备宽度来调整页面，达到适配终端大小的效果

自适应设计

  • meta标签的viewpoint属性

  • 不使用绝对宽度

  • 相对大小的字体：rem

  • 流动布局：float

  • 图片自适应
